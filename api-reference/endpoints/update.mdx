---
title: "PATCH /resources/{id} — Update a Resource"
sidebarTitle: "Update Resource"
description: "Update one or more fields on an existing resource. Only the fields you include are modified; omitted fields remain unchanged."
---

Use `PATCH /resources/{id}` to update an existing resource. The API uses partial updates — only the fields you include in the request body are changed. Fields you omit remain exactly as they were, so you never need to re-send the full resource object.

```text
PATCH https://api.google.com/v1/resources/{id}
```

## Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the resource to update (for example, `res_01J9KXPQ3N2M4R5T6W7Y8Z`).
</ParamField>

## Request body

All body fields are optional. Include only the fields you want to change.

<ParamField body="name" type="string">
  A new display name for the resource. Must be between 1 and 255 characters.
</ParamField>

<ParamField body="description" type="string">
  A new description for the resource. Pass an empty string (`""`) to clear an existing description.
</ParamField>

<ParamField body="status" type="string">
  The new status for the resource. Accepted values: `active`, `inactive`.
</ParamField>

<ParamField body="metadata" type="object">
  A set of key-value pairs to merge into the resource's existing metadata. New keys are added; existing keys are overwritten with the provided values. Keys are not deleted unless you explicitly pass a `null` value for them.
</ParamField>

## Example request

```bash
curl -X PATCH https://api.google.com/v1/resources/res_01J9KXPQ3N2M4R5T6W7Y8Z \
  -H "Authorization: Bearer $GOOGLE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"status": "inactive", "metadata": {"archived": "true"}}'
```

## Response

A successful request returns HTTP `200` with the full updated resource object nested under `data`.

<ResponseField name="data" type="object">
  The full resource object after applying your changes.

  <Expandable title="data fields">
    <ResponseField name="data.id" type="string">
      The unique identifier of this resource.
    </ResponseField>

    <ResponseField name="data.name" type="string">
      The display name of the resource.
    </ResponseField>

    <ResponseField name="data.description" type="string">
      The optional description attached to the resource.
    </ResponseField>

    <ResponseField name="data.status" type="string">
      The current status of the resource: `active` or `inactive`.
    </ResponseField>

    <ResponseField name="data.metadata" type="object">
      The full set of key-value pairs associated with this resource after the merge.
    </ResponseField>

    <ResponseField name="data.created_at" type="string">
      ISO 8601 timestamp for when the resource was created.
    </ResponseField>

    <ResponseField name="data.updated_at" type="string">
      ISO 8601 timestamp for when the resource was last updated (reflects this change).
    </ResponseField>
  </Expandable>
</ResponseField>

```json
{
  "data": {
    "id": "res_01J9KXPQ3N2M4R5T6W7Y8Z",
    "name": "My New Resource",
    "description": "Created via the API",
    "status": "inactive",
    "metadata": { "project": "demo", "archived": "true" },
    "created_at": "2024-01-15T10:30:00Z",
    "updated_at": "2024-01-15T11:45:00Z"
  },
  "meta": { "request_id": "req_ghi789" }
}
```

<Tip>
  Metadata updates are **merged**, not replaced. If your resource already has `{ "project": "demo" }` and you send `{ "archived": "true" }`, the result is `{ "project": "demo", "archived": "true" }`. To remove a specific key, include it in your request with a `null` value: `{ "project": null }`.
</Tip>

## Errors

| Status | Code | Description |
|--------|------|-------------|
| `400 Bad Request` | `invalid_parameter` | A field value is invalid (for example, `name` is empty or `status` is not `active` or `inactive`). Check the error message for the specific field. |
| `401 Unauthorized` | `authentication_required` | The `Authorization` header is missing or the Bearer token is invalid. Verify your API key and retry. |
| `404 Not Found` | `resource_not_found` | No resource exists with the provided ID. Verify the ID is correct and belongs to your account. |
