---

Use `GET /resources` to retrieve a paginated list of all resources in your account. You can narrow results by status, control the sort order, and page through large datasets using the `page` and `per_page` query parameters.

```text
GET https://api.google.com/v1/resources
```

## Query parameters

<ParamField query="page" type="integer" default={1}>
  The page number to retrieve. Starts at `1`.
</ParamField>

<ParamField query="per_page" type="integer" default={20}>
  The number of results to return per page. Maximum value is `100`.
</ParamField>

<ParamField query="status" type="string">
  Filter resources by status. Accepted values: `active`, `inactive`.
</ParamField>

<ParamField query="sort" type="string">
  The field to sort results by. Accepted values: `created_at`, `updated_at`, `name`.
</ParamField>

<ParamField query="order" type="string">
  The sort direction. Accepted values: `asc`, `desc`.
</ParamField>

## Example request

```bash
curl 'https://api.google.com/v1/resources?page=1&per_page=10&status=active' \
  -H "Authorization: Bearer $GOOGLE_API_KEY"
```

## Response

A successful request returns HTTP `200` with a JSON body containing a `data` array of resource objects and a `pagination` object.

<ResponseField name="data" type="array">
  The list of resource objects matching your query.
</ResponseField>

<ResponseField name="pagination" type="object">
  Metadata describing the current page and total result set.

  <Expandable title="pagination fields">
    <ResponseField name="pagination.page" type="integer">
      The current page number.
    </ResponseField>

    <ResponseField name="pagination.per_page" type="integer">
      The number of results returned on this page.
    </ResponseField>

    <ResponseField name="pagination.total" type="integer">
      The total number of resources matching your query.
    </ResponseField>

    <ResponseField name="pagination.total_pages" type="integer">
      The total number of pages available at the current `per_page` setting.
    </ResponseField>
  </Expandable>
</ResponseField>

## Errors

| Status | Code | Description |
|--------|------|-------------|
| `400 Bad Request` | `invalid_parameter` | A query parameter has an invalid value (for example, `per_page` exceeds `100` or `order` is not `asc` or `desc`). Check the error message for details. |
| `401 Unauthorized` | `authentication_required` | The `Authorization` header is missing or the Bearer token is invalid. Verify your API key and retry. |
