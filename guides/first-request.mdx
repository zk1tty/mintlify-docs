This guide walks you through crafting your first authenticated API request to Google. By the end, you'll have a working request, understand how to read the response, and know how to create your first resource — giving you a solid foundation to build on.

<Note>
  You need an API key to follow this guide. If you don't have one yet, head to the [Authentication](/authentication) page to generate your key before continuing.
</Note>

<Steps>
  <Step title="Set up your environment">
    Store your API key as an environment variable so you never hard-code credentials in your source code. Open a terminal and run:

    ```bash
    export GOOGLE_API_KEY="your_api_key_here"
    ```

    Add this line to your shell profile (e.g. `~/.bashrc` or `~/.zshrc`) to persist it across sessions. Your application code can then read it securely at runtime without exposing the key in version control.
  </Step>

  <Step title="Make the request">
    With your key in place, send a `GET` request to the `/v1/resources` endpoint. Pick the language you're most comfortable with:

    <CodeGroup>
      ```bash curl
      curl https://api.google.com/v1/resources \
        -H "Authorization: Bearer $GOOGLE_API_KEY" \
        -H "Content-Type: application/json"
      ```

      ```python Python
      import os, requests

      api_key = os.environ['GOOGLE_API_KEY']
      response = requests.get(
          'https://api.google.com/v1/resources',
          headers={'Authorization': f'Bearer {api_key}'}
      )
      print(response.json())
      ```

      ```javascript JavaScript
      const response = await fetch('https://api.google.com/v1/resources', {
        headers: {
          'Authorization': `Bearer ${process.env.GOOGLE_API_KEY}`,
          'Content-Type': 'application/json'
        }
      });
      const data = await response.json();
      console.log(data);
      ```
    </CodeGroup>
  </Step>

  <Step title="Read the response">
    A successful request returns an HTTP `200` with a JSON body structured like this:

    ```json
    {
      "data": [
        {
          "id": "res_01J9K...",
          "name": "Example Resource",
          "status": "active",
          "created_at": "2024-01-15T10:30:00Z"
        }
      ],
      "pagination": {
        "page": 1,
        "per_page": 20,
        "total": 143,
        "total_pages": 8
      }
    }
    ```

    The response contains two top-level fields:

    - **`data`** — An array of resource objects matching your query. Each object contains the resource's `id`, `name`, `status`, and timestamps.
    - **`pagination`** — Metadata about the full result set, including the current `page`, results `per_page`, `total` number of matching records, and `total_pages` available. Use these values to fetch additional pages when the result set is larger than a single response.
  </Step>

  <Step title="Create your first resource">
    To create a resource, send a `POST` request with a JSON body describing the new resource:

    ```bash
    curl -X POST https://api.google.com/v1/resources \
      -H "Authorization: Bearer $GOOGLE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "My First Resource",
        "status": "active"
      }'
    ```

    A successful creation returns an HTTP `201 Created` with the full resource object in the response body, including the server-assigned `id` and `created_at` timestamp. Save the `id` — you'll use it to read, update, or delete the resource in subsequent requests.
  </Step>
</Steps>
