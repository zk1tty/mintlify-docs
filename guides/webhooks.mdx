Webhooks let Google push event notifications to your server in real time, so you don't need to poll the API repeatedly to detect changes. Whenever a resource is created, updated, or deleted, Google sends an HTTP `POST` request to a URL you register — your server processes the payload and responds immediately.

## How webhooks work

The webhook lifecycle follows three steps:

1. **Register a URL** — You tell Google which URL to call and which events you care about.
2. **Google sends an HTTP POST** — When a matching event occurs, Google delivers a signed JSON payload to your endpoint.
3. **You respond with `200`** — Your server processes the event and returns an HTTP `200 OK` to acknowledge receipt. Any other status code — or no response within 5 seconds — is treated as a delivery failure.

## Register a webhook endpoint

Create a webhook by sending a `POST` request to `/v1/webhooks` with your endpoint URL and the list of events you want to subscribe to:

```bash
curl -X POST https://api.google.com/v1/webhooks \
  -H "Authorization: Bearer $GOOGLE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://yourapp.com/webhooks/google",
    "events": ["resource.created", "resource.updated", "resource.deleted"]
  }'
```

The API responds with the newly created webhook object, including its `id`. Store this `id` if you want to update or delete the webhook registration later.

## Available events

Subscribe to one or more of the following events when registering your webhook:

| Event               | When it fires                          |
|---------------------|----------------------------------------|
| `resource.created`  | A new resource is created              |
| `resource.updated`  | An existing resource is updated        |
| `resource.deleted`  | A resource is deleted                  |

You can subscribe to individual events or all of them in a single registration. To listen to every event, include all three values in the `events` array.

## Webhook payload

Every delivery shares the same envelope structure, with the event-specific data nested inside the `data` field:

```json
{
  "id": "evt_01J9K...",
  "type": "resource.created",
  "created_at": "2024-01-15T10:30:00Z",
  "data": {
    "id": "res_01J9K...",
    "name": "New Resource",
    "status": "active"
  }
}
```

- **`id`** — A unique identifier for this event delivery. You can use it to deduplicate events if your endpoint receives the same delivery more than once.
- **`type`** — The event name, matching one of the values in your subscription (e.g. `resource.created`).
- **`created_at`** — The ISO 8601 timestamp of when the event was generated.
- **`data`** — The full resource object at the time the event fired.

## Verify webhook signatures

Every webhook delivery includes an `X-Google-Signature` header containing an HMAC-SHA256 signature of the raw request body. Always verify this signature before processing the payload — it confirms the request genuinely came from Google and that the body hasn't been tampered with in transit.

To verify the signature, compute the HMAC-SHA256 of the raw request body using your webhook secret, then compare it to the value in the header:

```python
import hmac
import hashlib

def verify_signature(raw_body: bytes, secret: str, signature_header: str) -> bool:
    """Return True if the webhook signature is valid."""
    expected = hmac.new(
        key=secret.encode('utf-8'),
        msg=raw_body,
        digestmod=hashlib.sha256
    ).hexdigest()

    # Use compare_digest to prevent timing attacks
    return hmac.compare_digest(expected, signature_header)


# Example usage inside a Flask route
import os
from flask import Flask, request, abort

app = Flask(__name__)
WEBHOOK_SECRET = os.environ['GOOGLE_WEBHOOK_SECRET']

@app.route('/webhooks/google', methods=['POST'])
def handle_webhook():
    signature = request.headers.get('X-Google-Signature', '')
    if not verify_signature(request.get_data(), WEBHOOK_SECRET, signature):
        abort(403)  # Reject requests with invalid signatures

    event = request.get_json()
    # Process the event...
    return '', 200
```

You can find your webhook secret in the Google dashboard under **Settings → Webhooks**. Rotate it immediately if you suspect it has been compromised.

## Respond to webhooks

Your endpoint must return an HTTP `200 OK` within **5 seconds** of receiving a delivery. If your handler needs to perform slow operations — like writing to a database or calling a third-party API — acknowledge the webhook immediately and process the work asynchronously in a background job:

```python
import os
from threading import Thread
from flask import Flask, request, abort

app = Flask(__name__)
WEBHOOK_SECRET = os.environ['GOOGLE_WEBHOOK_SECRET']

def process_event(event):
    # Add your event-handling logic here
    pass

@app.route('/webhooks/google', methods=['POST'])
def handle_webhook():
    # Verify signature first (see the section above)
    signature = request.headers.get('X-Google-Signature', '')
    if not verify_signature(request.get_data(), WEBHOOK_SECRET, signature):
        abort(403)

    event = request.get_json()

    # Kick off background processing and respond at once
    Thread(target=process_event, args=(event,), daemon=True).start()
    return '', 200
```

<Warning>
  If your endpoint fails to respond with `200` — either by returning an error status or timing out — Google will retry the delivery up to **5 times** using exponential backoff. Make your handler idempotent so that processing the same event twice doesn't corrupt your data. Use the `id` field on the payload to detect and safely skip duplicate deliveries.
</Warning>
